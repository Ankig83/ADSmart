# Остановки — расхождения с Figma (140:5549, 140:5560, 140:5588, 140:7415)

## Найденные проблемы

1. **Фиксированные ширины**
   - `.demo-stops` и все секции: `width: 1440px` без `max-width` → overflow на экранах <1440px.

2. **Типографика / цвет по Figma**
   - Benefits (140:5569, 140:5575, 140:5581, 140:5587): текст карточки в Figma `rgba(255,255,255,0.8)`, в коде — `#ffffff`.
   - Poster (140:5594, 140:5598, …): заголовок карточки в Figma `#ff7b00`, в десктопном CSS — `#ffffff`.

3. **Отступы**
   - Hero кнопки (140:5557): в Figma gap между кнопками 11px, в коде 14px.

4. **Контент формы (834px)**
   - Заголовок формы в макете 834px: «Получить комплексное предложение» (140:7427), в HTML — «Обсудим ваш проект?».

5. **Дублирование правил**
   - В `stops.css` два блока `@media (min-width: 834px) and (max-width: 1439px)`: один только contact (стр. 12–188), второй — полный (стр. 589+). Плюс `stops-tablet-lock.css`. Конфликты и избыточные переопределения.

6. **Poster карточки (mobile)**
   - В mobile-lock border карточки 1px, в Figma 2px.

7. **Hero overlay**
   - Figma (140:5553): gradient `4.66%, 36.68%, 59.73%`; в коде `0%, 58.15%, 100%` — небольшие отличия.

8. **Contact form title (desktop)**
   - В макете форма справа может иметь заголовок «Получить комплексное предложение»; в HTML везде «Обсудим ваш проект?» — приведено к Figma для 834 и консистентности.
