## Стратегия работы

### 1. Архитектура медиа-запросов
Использовать каскадную стратегию для предотвращения конфликтов:
```css
/* Базовые стили (mobile-first) */
.element { ... }

/* 834-1439px (tablet) */
@media (min-width: 834px) and (max-width: 1439px) {
  .element { ... }
}

/* 1440px+ (desktop) */
@media (min-width: 1440px) {
  .element { ... }
}
```

### 2. Источники данных из Figma
- **Основной источник**: `design/responsive-home.json` (уже экспортирован)
- **MCP сервер**: Использовать для получения актуальных данных из Figma
- **Node IDs**: Использовать существующие ID из Figma для точного позиционирования

## Методология работы с MCP и Figma

### Шаг 1: Получение данных из Figma
```bash
# Использовать MCP сервер для получения данных
# Или использовать существующий JSON файл
# Или экспортировать через скрипт figma_export.py
```

### Шаг 2: Извлечение метрик
Для каждого элемента извлечь:
- `absoluteBoundingBox` (x, y, width, height)
- `absoluteRenderBounds` (для точного позиционирования)
- `padding` (paddingLeft, paddingRight, paddingTop, paddingBottom)
- `itemSpacing` (gap между элементами)
- `fontSize`, `lineHeight`, `letterSpacing`
- `color` (fills)
- `effects` (blur, shadows)

### Шаг 3: Расчет позиций
- Вычислить отступы между секциями: `nextSection.y - (prevSection.y + prevSection.height)`
- Учесть padding при расчете внутренних отступов
- Преобразовать абсолютные координаты в относительные (относительно родителя)

### Шаг 4: Применение стилей
- Использовать `!important` для переопределения базовых стилей
- Применять каскадную стратегию медиа-запросов
- Комментировать каждый блок с указанием Node ID из Figma
